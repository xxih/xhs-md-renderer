# cli配置目录与溢出检测

## 背景

当前 CLI 只能靠命令行参数一次性拼配置，头像也只能传入运行时字符串，缺少一个稳定、可约定的配置读取目录。后续如果要让 AI 持续调用 CLI，就需要一个固定位置来放渲染配置和头像素材，而不是每次临时组装参数。

同时，当前渲染链路没有“页面底部安全区”和“溢出反馈”能力。CLI 虽然能产出图片，但无法告诉调用方某一页文字是否已经贴底、是否越过安全边距、该往哪个方向调整。

## 本轮目标

1. 为 CLI 增加固定配置目录约定，并支持从配置文件读取渲染参数。
2. 让头像素材可以通过配置目录中的文件稳定参与渲染，而不是只能依赖手写 data URL。
3. 在核心渲染配置中引入正文底部安全边距，并为每页生成布局反馈。
4. 让 CLI 在控制台和输出目录中都能提供溢出/贴底报告，供后续 AI 调整内容或配置。

## 约束

1. 不能破坏现有 `--input --output` 基本链路。
2. 固定配置目录要有明确、可记忆的默认规则，同时允许显式覆盖。
3. 溢出检测优先提供稳定、可自动消费的反馈，不追求浏览器像素级测量。
