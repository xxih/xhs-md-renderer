# 规格说明：cli配置目录与溢出检测

## 1. 背景

CLI 目前只能通过命令行参数零散注入配置，没有固定配置目录，也没有稳定的头像素材读取约定。对于批处理和 AI 驱动调用，这种模式不够稳定，因为每次都要重新拼参数、重新传头像来源。

此外，当前渲染结果没有布局反馈。即使页面文字已经过长、离底部太近，CLI 也不会输出任何结构化信号，导致后续自动调节只能靠猜。

## 2. 交付目标

### 2.1 固定配置目录

系统必须为 CLI 约定一个固定的默认配置目录名 `.xhs-md-renderer`。CLI 在未显式传入配置目录时，必须从输入 Markdown 所在目录开始向上查找该目录；若未找到，则继续按默认配置运行，并明确告知“未找到配置目录”。

成功标志：调用方只要在项目目录中放置 `.xhs-md-renderer/`，CLI 就能稳定读取同一套配置。

### 2.2 配置文件读取与显式覆盖

系统必须支持从配置目录中的 `render.json` 读取渲染配置。配置文件至少可以覆盖：

- 文档标题
- 主题
- 字体
- 字号
- 身份信息与各类显隐开关
- 正文底部安全边距与预警阈值

CLI 仍必须保留现有命令行参数，并采用“CLI 参数优先于配置文件”的覆盖顺序。

成功标志：用户既可以只维护配置文件，也可以在单次运行中用 CLI 参数临时覆盖局部字段。

### 2.3 头像素材稳定读取

系统必须支持在配置目录中稳定读取头像素材。支持两种方式：

1. `render.json` 中显式声明相对配置目录的头像文件路径。
2. 未声明时，自动尝试读取配置目录根下默认命名的头像文件。

CLI 在渲染前必须把头像文件解析成可用于统一渲染链路的图片源，避免要求调用方自行构造 data URL。

成功标志：调用方只需在固定目录放头像文件并配置路径，CLI 就能稳定把头像渲染进结果。

### 2.4 页面底部安全边距

系统必须在统一渲染配置中提供正文底部安全边距配置，并让页面正文区域保留该安全边距，而不是允许正文文字直接贴到底部。

成功标志：默认渲染下，每页正文区域底部留有固定缓冲区；即使无 footer，也不会出现正文紧贴底部边缘的排版。

### 2.5 溢出与贴底反馈

系统必须为每一页生成布局分析结果，至少区分以下状态：

- `ok`：剩余底部空间高于安全边距预警阈值
- `warning`：未溢出，但剩余底部空间低于安全边距预警阈值
- `overflow`：正文估算高度超过可用内容区

反馈结果必须同时：

1. 写入输出目录中的结构化 JSON 报告。
2. 在 CLI 控制台输出摘要，明确指出哪一页需要调整。

成功标志：后续 AI 可以直接读取报告，知道是“贴底预警”还是“已经溢出”。

## 3. 非目标

- 本轮不做自动重新分页。
- 本轮不接入云端配置同步。
- 本轮不做 Web 端溢出提示 UI。
- 本轮不追求浏览器级精确布局测量。

## 4. 关键约束

- 配置目录规则必须稳定、可文档化、可自动发现。
- 渲染配置的新增字段必须进入 `core`，不能只留在 CLI 私有对象里。
- 溢出反馈必须是结构化数据，不能只有人读得懂的日志。

## 5. 验收标准

1. CLI 能自动发现 `.xhs-md-renderer/render.json` 并应用配置。
2. CLI 支持通过配置目录中的头像文件稳定渲染头像。
3. 页面正文底部保留安全边距，不再默认贴底。
4. 输出目录新增布局报告文件。
5. CLI 控制台会输出每页 `ok / warning / overflow` 摘要。
