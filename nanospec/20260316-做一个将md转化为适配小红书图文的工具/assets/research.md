# 调研记录

调研时间：2026-03-16

## 1. 参考库 `note-to-red` 的能力边界

本地参考库 `assets/note-to-red` 本质上是一个 Obsidian 桌面插件，核心价值是把一篇 Markdown 笔记拆分成多页“小红书风格图片”。

从代码和 README 看，它当前已经覆盖了这些能力：

- 以标题层级作为分页依据，把一篇笔记拆成多页图文内容。
- 在预览区实时渲染，并支持锁定预览、翻页查看。
- 支持模板、主题、字体、字号、头像、昵称、页脚等视觉定制。
- 支持单页导出和批量导出 ZIP。

它的内部结构也比较清晰：

- 先把 Markdown 渲染成 DOM，再做分段和样式处理，而不是直接操作 Markdown AST。
- “结构模板”和“视觉主题”是分层的，前者决定页面骨架，后者决定样式细节。
- `manifest.json` 标记了 `isDesktopOnly: true`，说明它一开始就是本地桌面创作辅助，而不是服务化流水线。

它当前没有覆盖这些能力：

- 选题生成、标题生成、文案改写、标签推荐。
- 从原始素材自动提炼成小红书文案结构。
- 发布包管理、版本管理、审核流。
- 自动发帖、账号运营、数据回收。

另外，它虽然解决了出图，但自动化深度仍然有限：

- 长文分页仍然需要用户通过标题和 `---` 手工控制。
- 首图、字号、长页内容压缩等仍然主要靠人工判断和调整。
- 更像“可视化排版器”，还不是“内容编排器”。

结论：`note-to-red` 更像“最后一公里的图文排版导出器”，而不是完整的创作自动化工具。

## 2. 外部现状分层

结合外部项目，当前“小红书创作自动化”大致分成三层：

### 2.1 排版导出层

代表：`note-to-red`

- 解决“已有内容怎么变成适合发的小红书图文”。
- 重点是分页、模板、视觉风格、导出。
- 优点是可控、低风险、容易本地化。
- 缺点是只解决最后一步，不能减少前面的构思和改写工作。

### 2.2 平台发布层

代表：`xhs-toolkit`

- 主打 Markdown/HTML 到小红书等平台的自动发布。
- 常见实现依赖 cookie、浏览器自动化、chromedriver 或平台页面结构。
- 优点是省掉手工发布。
- 缺点是环境脆弱、维护成本高、平台改版易失效，也更容易碰到账号安全和合规问题。

### 2.3 AI 创作流水线层

代表：`Xiaohongshu-AI-Tool`

- 重点是选题、标题、爆款分析、文案生成、图片生成、矩阵运营等。
- 优点是覆盖面广，适合“从 0 到 1”生成。
- 缺点是范围很大，容易把产品做成泛 AI 写作壳；如果没有好的人工控制点，产出质量和可复用性都不稳定。

## 3. 可借鉴机会

调研后更值得做的，不是直接复制任何一层，而是把中间那段补齐：

- 输入一份已有素材：笔记、草稿、文章大纲。
- 自动整理成适合小红书的结构：封面钩子、分页逻辑、正文段落、结尾 CTA、标签组。
- 允许用户逐段修改和锁定。
- 一次性导出完整发布包，而不是只导出图片。

这个方向比“只做 Markdown 转图文”更有价值，也比“直接自动发布”更稳。

## 4. 推荐方向

推荐把 v1 定位成“小红书创作装配台”：

- 上游接收 Markdown / 笔记素材。
- 中间完成结构改写、分页、标题候选、正文候选、标签候选。
- 下游导出一个可人工审核的发布包，包含图文页、文案、标题、标签和发布检查清单。

这样既能吸收 `note-to-red` 的图文导出能力，又不会一上来就把范围推到高风险的自动发帖。

## 6. 实现选型结论

本轮进入实现前，额外做了一次 `search-first` 取舍，结论如下：

### 6.1 adopt

- `React 19 + Vite 7` 作为 Web 侧基础栈
- `unified + remark-parse + remark-gfm` 作为 Markdown 解析骨架
- `satori + @resvg/resvg-js` 作为 Node 侧统一出图链路

### 6.2 adapt

- 复用 `note-to-red` 的核心思路，而不是直接复用它的 Obsidian 插件结构
- 重点吸收：
  - 标题切页
  - `---` 手动分页
  - 模板和主题分层
  - 预览与导出的分离

### 6.3 build

以下部分自行构建，不直接照搬参考库：

- 统一 `core` 的分页数据模型
- `CLI / Web / Obsidian` 共享的接口边界
- `export-bundle` 协议
- 末端工具仓库结构

当前决策是 `adapt + build`，不是 `adopt` 整个 `note-to-red`。
