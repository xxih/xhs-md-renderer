# 做一个将 md 转化为适配小红书图文的工具

## 背景

用户想做一个工具，自动化小红书创作过程中的部分环节，但当前还没有确定最终要做成什么。任务目录中已放入 `assets/note-to-red` 作为参考库，希望先仔细调研，再产出一版可对齐的 spec。

## 当前目标

1. 研究 `note-to-red` 当前解决了什么问题、采用了什么交互和产物形态。
2. 补一轮外部现状调研，判断“小红书创作自动化”现有工具主要集中在哪些环节。
3. 写出围绕统一 `core`、CLI / Web / Obsidian 多端复用的“md 转图文末端工具”规格，而不是整条创作工作流规格。
4. 补一版高层 `outputs/2-plan.md`，明确这个末端工具的核心引擎、渲染模块和适配层边界。

## 当前约束

- 这轮以调研和规格收敛为主，不进入实现。
- 不默认把“自动发帖”当成目标，除非后续明确确认。
- 优先考虑可复用现有 Markdown / 笔记资产的方向。
- 未来整体会接入 `script + skill` 工作流，但本轮只聚焦末端图文工具本身。
