# 规格说明：markdown图片正确渲染

## 1. 背景

当前仓库已经能在 Markdown 解析阶段识别图片节点，但共享渲染层仍把它们输出为占位块，导致 Web 预览、CLI 导出和布局反馈都没有真正支持 Markdown 图片。

这不是单一界面问题，而是一个横跨 `packages/core`、`packages/cli` 和 `apps/web` 的共享渲染缺口：图片节点已经存在，但资源解析、可视渲染和布局估算还没有串起来。

## 2. 交付目标

### 2.1 标准 Markdown 图片语法输出真实图片

系统必须把标准 Markdown 图片语法 `![alt](src)` 渲染为真实图片区块，而不是占位块。该能力必须进入共享渲染链路，使 Web 预览与导出结果基于同一套图片渲染语义工作。

成功标志：同一份包含 Markdown 图片的内容，在 Web 预览和导出产物中都能看到真实图片，不再出现 `Image Placeholder`。

### 2.2 按运行环境解析可用图片源

系统必须明确图片源的解析边界，并按运行环境提供可用资源：

- CLI / Node 导出链路：支持相对于 Markdown 文件的本地图片路径，以及可直接读取的绝对路径、`http(s)` URL、`data:` URL。
- Web 编辑预览链路：至少支持浏览器可直接访问的 `http(s)` URL 和 `data:` URL；对于浏览器当前无法直接读取的本地相对路径，必须以稳定、可理解的失败态呈现，而不是静默假装渲染成功。

成功标志：CLI 能渲染本地相对图片文件；Web 对浏览器可访问图片源能直接显示，对不可访问图片源有明确失败表现。

### 2.3 图片区块版式正确，不拉伸、不越界

系统必须让图片在正文内容区内按统一版式显示，至少满足：

- 不拉伸变形，保持原始宽高比；
- 不超出页面正文内容区；
- 与正文区块共享稳定的边距、圆角和块级间距口径；
- 图片成功加载时，不把 `alt` 文本默认当作可见 caption 渲染。

成功标志：大图、长图和普通横图都不会被错误拉伸或挤出卡片边界；`alt` 文本仅用于语义和失败回退，不会默认额外占一行说明文字。

### 2.4 图片进入共享布局估算与反馈

系统必须让图片区块参与页面布局估算，而不是继续使用固定占位高度。布局估算可以基于已解析到的图片尺寸信息或稳定的回退口径，但不能继续把所有图片都当成同一块固定高度的占位区域。

成功标志：带图片页面的布局反馈不再沿用当前固定 `140px` 占位口径，图片大小变化会反映到页面剩余空间和 `ok / warning / overflow` 结果中。

### 2.5 导出链路与预览链路口径一致

系统必须保证共享页面模型、共享渲染结构与图片资源处理口径在预览和导出中保持一致，避免出现“Web 能看到，CLI 导不出”或“CLI 导出了，Web 仍是占位图”的分叉。

成功标志：同一份 Markdown 内容在 Web 预览与 CLI 导出中，图片是否显示、显示比例和失败回退语义保持一致。

## 3. 非目标

以下内容不属于本轮：

- HTML `<img>` 标签支持
- 引用式图片语法与尺寸属性扩展
- 多图拼版、轮播、图库管理
- 自动重新分页或自动裁切图片
- 封面专属图片模板系统

## 4. 关键约束

1. 图片能力必须建立在当前共享 `core` 模型之上，不能让 Web 和 CLI 各自维护一套独立图片渲染逻辑。
2. CLI 与 Web 的图片源能力可以不同，但差异必须被显式建模，不能隐式失败。
3. 已有文本、列表、代码块和页眉页脚渲染能力不能因图片接入而回退。
4. 本轮不引入后端服务或远端素材托管作为前置条件。

## 5. 验收标准

1. Web 预览中的 Markdown 图片不再显示为占位块。
2. CLI 导出包含本地相对图片路径的 Markdown 时，导出的 PNG 中能看到真实图片。
3. 图片在页面正文区中保持宽高比，不拉伸、不越界。
4. 图片成功加载时，`alt` 文本不会默认作为可见 caption 占位显示。
5. 带图片页面的布局反馈不再沿用固定占位高度，而会反映图片的有效尺寸影响。
