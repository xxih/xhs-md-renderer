# 对齐记录

## 2026-03-16

- [变更] 本任务从上一轮“统一 core 的 md 转图文工具”中拆出独立执行面，专门处理样式对齐与配置补齐，不再继续展开更高层工作流设计。
- [偏差] 当前实现的配置项只覆盖 `theme + splitHeadingLevel + name/handle/footer`，与 `note-to-red` 已有的常用外观配置存在明显缺口，需要先补齐统一 `core` 模型再谈多端体验一致性。
- [变更] 本轮样式对齐目标收敛为 `note-to-red` 的默认主题结构与视觉语言，而不是一次性支持其全部预设主题。
- [变更] 本轮配置补齐范围收敛为 `字体、字号、标题文案、用户信息、时间显示、页脚开关、页脚左右文案`，并要求 Web 与 CLI 共用同一份配置模型。
- [缺失] 头像图片上传、背景图上传/拖拽、完整主题编辑器与主题显示管理暂不纳入本轮实现，避免把任务膨胀成整套配置系统重写。
- [偏差] 用户反馈当前导出图的尺寸比例仍未对齐，主要表现为页眉信息区过小。已核对 `note-to-red` 默认模板的 `40px` 头像、`16px` 用户名、`13px` 时间与 `450px` 预览宽度的比例，确认我们当前 `1080px` 导出宽度下的页眉/页脚尺寸偏小，需要按同一比例重新标定。
- [变更] 用户提供了新的成品参考图，并明确要求按 `1800x2400` 的画布尺寸和对应比例调整现有实现。因此当前默认导出尺寸从 `1080x1440` 上调到 `1800x2400`，并要求 Web 预览端同步改成按真实画布比例缩放，而不是继续写死旧尺寸。
- [变更] Web 预览端本轮不再只做“能看”，而是要围绕参考图做一版更像实际创作台的视觉外壳：深色预览舞台、动态画布缩放、较少的 demo 感。
- [偏差] 用户进一步明确：当前问题不在 Web 体验，而在产出物的文字比例。对照新参考图后，确认现有导出图的头像、昵称、日期等页眉信息仍然偏小；需要按参考图重新标定导出物的 header 尺寸，而不是继续优化预览壳子。
