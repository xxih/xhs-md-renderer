# 方案：样式对齐与选项补齐

## 1. 先收敛配置模型，再改三端

先从 `packages/core/src/models.ts` 和 `packages/core/src/themes.ts` 扩展 `RenderConfig`。这一步负责定义本轮新增的外观字段和默认值，避免 Web、CLI 各自引入临时参数口径。

## 2. 用默认主题完成第一次视觉对齐

在 `packages/core/src/preview.tsx` 中重构页面结构和样式，把当前页面改成更接近 `note-to-red` 默认主题的深色图文卡片。这里不实现主题编辑器，只先保证默认风格、排版节奏和信息区结构足够接近。

## 3. Web 端补齐配置表单

在 `apps/web/src/App.tsx` 中接入新增配置项，并整理控制面板结构。目标不是堆更多输入框，而是把“身份信息、排版、页脚、日期”几个配置组明确分开。

## 4. CLI 补齐参数映射

在 `packages/cli/src/index.ts` 中补齐参数解析，把命令行输入完整映射到统一 `RenderConfig`。同时更新帮助文案，确保用户能看懂新的可选项。

## 5. 验证路径

验证分两层：

1. 跑仓库构建，确认 `core + cli + web` 全部通过。
2. 跑一条 CLI 实际导出，确认新增配置已进入最终图片产物。
3. 以用户提供的 `1800x2400` 参考图为基准，检查导出尺寸和 Web 预览缩放是否已经同步。
