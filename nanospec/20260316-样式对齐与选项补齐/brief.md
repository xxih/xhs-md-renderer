# 样式对齐与选项补齐

## 背景

用户要求为 `xhs-md-renderer` 开一个新任务，先把当前渲染样式向
`nanospec/20260316-做一个将md转化为适配小红书图文的工具/assets/note-to-red`
参考实现对齐，再把当前明显缺失的渲染配置项补齐。

## 当前目标

1. 让 `packages/core` 的页面视觉更接近 `note-to-red` 默认主题的结构和氛围。
2. 为统一 `core` 补齐一批 v1 级别的渲染配置项，并同步贯通到 Web 与 CLI。
3. 保持“同一份配置模型，多端复用”的方向，不把样式逻辑重新分散回各端。

## 当前约束

- 本轮以“默认主题样式对齐 + 常用配置补齐”为主，不重做完整主题系统。
- 参考 `note-to-red` 的布局和配置口径，但不直接搬运其整套主题编辑器与背景图流程。
- 需要继续沿用仓库中现有 `core + cli + web` 架构。
- `assets/note-to-red` 是外部参考仓库，不纳入当前仓库管理，也不修改其内容。
