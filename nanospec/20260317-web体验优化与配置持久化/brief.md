# web体验优化与配置持久化

## 背景

用户要求为 `xhs-md-renderer` 单独开一个新任务，聚焦当前 Web 端的编辑与预览体验问题，而不是继续把范围混在前一轮样式对齐任务里。

当前已明确的问题与新增要求包括：

- Web 界面需要中文优先，而不是继续保留明显偏英文的控制面板。
- 左侧配置面板过于松散，配置起来费劲，需要压缩信息密度，尽可能在一个屏幕里看全常用配置。
- 右侧 page 预览当前存在明显偏移，整体观感较差，需要修正预览舞台的布局与缩放策略。
- Web 端需要能直接导出图片包，而不只是下载 `pages.json`。
- 默认状态不要显示 footer。
- 页眉需要补上参考样式里头像旁边的蓝色认证标志。
- 所有非正文元素都需要支持开关控制。
- 头像需要可配置，并且该配置需要持久化。

## 当前目标

1. 把 Web 控制面板改成中文优先、紧凑、易扫读的编辑器界面。
2. 修正右侧预览舞台的缩放和定位，让多页预览稳定、居中、不漂移。
3. 在统一渲染模型中补齐头像、认证标志和各类非正文元素的开关能力。
4. 为 Web 增加直接导出图片包的能力。
5. 让 Web 中与头像相关的配置可以持久化，避免每次刷新都重填。

## 当前约束

- 正常页面仍然按 Markdown 渲染，不把封面定制能力混进正文渲染。
- 如果未来要做封面，必须走单独配置，不复用本轮的正文页面配置。
- 当前仓库仍以 `packages/core + packages/cli + apps/web` 为主架构，不引入后端服务作为前提。
- 配置持久化默认优先采用本地浏览器存储，除非后续另行对齐。
